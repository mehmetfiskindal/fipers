# Code of Conduct

## Our Pledge

In the interest of fostering an open, professional, and security-focused community, we as contributors and maintainers of **FIPERS** pledge to make participation in this project a respectful and constructive experience for everyone.

This project prioritizes **technical excellence, security, and correctness**. Healthy disagreement is expected; disrespectful behavior is not tolerated.

---

## Our Standards

Examples of behavior that contributes to a positive environment include:

* Using welcoming, professional, and respectful language
* Focusing on technical merits rather than individuals
* Providing constructive and evidence-based feedback
* Accepting criticism gracefully and responding with reasoned arguments
* Respecting differing viewpoints and experience levels

Examples of unacceptable behavior include:

* Personal attacks, insults, or derogatory comments
* Harassment, intimidation, or deliberate disruption
* Dismissive or belittling remarks unrelated to technical content
* Publishing private information without explicit consent
* Repeatedly ignoring project guidelines or maintainer decisions

---

## Security-Focused Communication

Given the nature of this project:

* Security concerns must be discussed responsibly
* Potential vulnerabilities should **not** be publicly exploited or sensationalized
* Contributors are expected to follow responsible disclosure practices

If in doubt, contact the maintainers privately.

---

## Scope

This Code of Conduct applies within all project spaces, including but not limited to:

* GitHub repositories (issues, pull requests, discussions)
* Code reviews and design discussions
* Community communication channels related to the project

---

## Enforcement

Project maintainers are responsible for clarifying standards and enforcing this Code of Conduct.

Maintainers may take any action deemed appropriate, including:

* Requesting changes to communication or behavior
* Issuing warnings
* Temporarily or permanently removing contributions
* Restricting or revoking participation privileges

---

## Reporting

If you experience or witness behavior that violates this Code of Conduct, please report it by contacting the project maintainers.

Reports should include:

* A clear description of the incident
* Relevant context (links, screenshots, commit hashes)
* Any steps already taken

All reports will be handled discreetly and reviewed objectively.

---

## Maintainer Responsibility

Maintainers commit to:

* Acting in good faith
* Evaluating situations based on facts and context
* Prioritizing project integrity, security, and collaboration

---
